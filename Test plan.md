# Введение

## Цель

1) Функциональное тестирование сайта: убедится, что сайт и все его ключевые функции работают согласно спецификации.
2) Тестирование производительности: убедиться, что сайт и его функции работают эффективно под определённой нагрузкой
3) Тестирование безопасности: выявление уязвимости в системы безопасности, проверка авторизации и смежных систем
4) Кроссбраузерное тестирование: проверка корректного отображения сайта в зависимости от браузера
5) Выявление багов и критических дефектов
6) Мобильное тестирование 

# Область тестирования

В области тестирования находятся:
1. Ручное тестирование основных endpoint's saucedemo.com /saucedemo.com/inventory.html / saucedemo.com/cart.html/ saucedemo.com/checkout-step-one.html /saucedemo.com/checkout-step-two.html / saucedemo.com/checkout-complete.html
2. Область авторизации пользователя
3. Область Products ( переход по страницам с товаром), рабочие кнопки add to cart / remove, сортировка по цене/названию
4. Область cart( рабочий функционал кнопок (remove,continue shopping, checkout))
5. Страницы checkout для подтверждения покупок, и их модули (кнопки, поля заполнения, уведомление о успешной покупке)
6. Боковая панель и её кнопки

# Цели тестов

1. Проверка существующих endpoint's, связанных с сайтом и описанных в области тестирования
2. Проверка аутентификации входа и выхода из системы
3. Проверка функциональности страниц существующих товаров
4. Проверка функциональности системы подтверждения покупки, заполнения форм, уведомление о приобретении
5. Проверка функциональности корзины, добавление и удаление товаров.
6. Проверка функциональности боковой панели

# Подход к тестированию

## Базовые метрики

## Процентный статус по выполнению созданных тест кейсов:

1. Общее количество тест-кейсов
2. Общее количество выполненных тест-кейсов

Процент выполненных тест-кейсов = (Количество выполненных тест-кейсов / Общее количество написанных тест-кейсов) x 100


## Эффективность тест-кейсов (Test Case Effectiveness):
Эффективность тест-кейсов = (Количество обнаруженных дефектов / Количество выполненных тест-кейсов) x 100

## Процент прошедших тест-кейсов (Passed Test Cases Percentage) — показывает процент успешно выполненных тест-кейсов.

Процент прошедших тест-кейсов = (Общее количество прошедших тест-кейсов / Общее количество выполненных) x 100

## Процент упавших тест-кейсов (Failed Test Cases Percentage): показывает процент упавших/красных тест-кейсов:

Процент упавших тест-кейсов = (Общее количество упавших тест-кейсов / Общее количество выполненных) x 100

## Процент блокированных тест-кейсов (Blocked Test Cases Percentage): показывает процент заблокированных тест-кейсов:

Процент блокированных тест-кейсов = (Количество заблокированных тест-кейсов / Общее количество выполненных) x 100
